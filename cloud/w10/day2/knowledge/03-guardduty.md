> **Nguồn:** W10-D2 | **Chủ đề:** AWS GuardDuty

## 1. GuardDuty là gì?

**GuardDuty** là managed threat detection service — phân tích **log data** từ nhiều nguồn để phát hiện **hành vi đáng ngờ** trong AWS account.

```
Data Sources:
  ┌─────────────────┐  ┌──────────────────┐  ┌───────────────┐
  │  CloudTrail      │  │  VPC Flow Logs   │  │  DNS Logs     │
  │  Management +    │  │  (network traffic│  │  (DNS queries │
  │  Data Events     │  │   patterns)      │  │   from VPC)   │
  └────────┬─────────┘  └────────┬─────────┘  └───────┬───────┘
           │                     │                     │
           ▼                     ▼                     ▼
  ┌────────────────────────────────────────────────────────────┐
  │                    AWS GuardDuty                            │
  │                                                            │
  │  Machine Learning + Threat Intelligence + Anomaly Detection│
  │                                                            │
  │  → Findings (severity: Low / Medium / High / Critical)     │
  └────────────────────────────────────────────────────────────┘
```

### Đặc điểm:

- **Zero infrastructure** — không cần cài agent, deploy sensor
- **ML-powered** — phát hiện anomalies tự động
- **Threat intelligence** — AWS feeds + 3rd party (CrowdStrike, Proofpoint)
- **One-click enable** — bật 1 nút, bắt đầu detect
- **Pay-per-volume** — theo lượng data analyzed

## 2. Finding Types

GuardDuty phân loại findings theo:

### Reconnaissance (Thăm dò)

| Finding | Mô tả |
|---|---|
| `Recon:EC2/PortProbeUnprotectedPort` | EC2 port bị scan từ bên ngoài |
| `Recon:EC2/Portscan` | EC2 đang scan ports (compromised?) |

### Unauthorized Access (Truy cập trái phép)

| Finding | Mô tả |
|---|---|
| `UnauthorizedAccess:EC2/SSHBruteForce` | SSH brute force attack |
| `UnauthorizedAccess:IAMUser/ConsoleLoginSuccess.B` | Console login từ IP lạ |
| `UnauthorizedAccess:IAMUser/InstanceCredentialExfiltration` | EC2 credentials dùng từ ngoài AWS |

### Trojan / Malware

| Finding | Mô tả |
|---|---|
| `Trojan:EC2/BlackholeTraffic` | EC2 gửi traffic đến black hole IP |
| `Trojan:EC2/DGADomainRequest.B` | EC2 query domain generated by DGA (Domain Generation Algorithm) |

### Cryptocurrency

| Finding | Mô tả |
|---|---|
| `CryptoCurrency:EC2/BitcoinTool.B` | EC2 đang mine crypto |
| `CryptoCurrency:Runtime/BitcoinTool.B` | Container đang mine crypto |

### Severity levels:

```
Critical (9.0-10.0): Immediate action required
  → EC2 credentials exfiltrated
  
High (7.0-8.9): Urgent investigation
  → SSH brute force success, crypto mining
  
Medium (4.0-6.9): Investigate soon
  → Unusual API calls, port scanning
  
Low (1.0-3.9): Monitor & review
  → Reconnaissance attempts
```

## 3. Bật GuardDuty bằng Terraform

```hcl
# guardduty.tf

# Enable GuardDuty
resource "aws_guardduty_detector" "main" {
  enable = true
  
  # Bật tất cả data sources
  datasources {
    s3_logs {
      enable = true
    }
    kubernetes {
      audit_logs {
        enable = true
      }
    }
    malware_protection {
      scan_ec2_instance_with_findings {
        ebs_volumes {
          enable = true
        }
      }
    }
  }
  
  # Finding publishing frequency
  finding_publishing_frequency = "FIFTEEN_MINUTES"
  
  tags = {
    Environment = var.environment
  }
}
```

## 4. Alert Integration — GuardDuty → SNS

Dùng **EventBridge** (không phải SNS trực tiếp) để route GuardDuty findings:

```hcl
# EventBridge rule cho GuardDuty findings
resource "aws_cloudwatch_event_rule" "guardduty_findings" {
  name        = "guardduty-high-severity"
  description = "Alert on high severity GuardDuty findings"
  
  event_pattern = jsonencode({
    source      = ["aws.guardduty"]
    detail-type = ["GuardDuty Finding"]
    detail = {
      severity = [
        { numeric = [">=", 7] }    # High + Critical
      ]
    }
  })
}

# Route to SNS
resource "aws_cloudwatch_event_target" "guardduty_sns" {
  rule      = aws_cloudwatch_event_rule.guardduty_findings.name
  target_id = "guardduty-to-sns"
  arn       = aws_sns_topic.security_alerts.arn
}

# SNS Topic cho security alerts
resource "aws_sns_topic" "security_alerts" {
  name = "security-alerts-${var.environment}"
}

resource "aws_sns_topic_subscription" "security_email" {
  topic_arn = aws_sns_topic.security_alerts.arn
  protocol  = "email"
  endpoint  = "security-team@company.com"
}

# Cho phép EventBridge publish to SNS
resource "aws_sns_topic_policy" "guardduty" {
  arn = aws_sns_topic.security_alerts.arn
  
  policy = jsonencode({
    Version = "2012-10-17"
    Statement = [
      {
        Sid    = "AllowEventBridge"
        Effect = "Allow"
        Principal = {
          Service = "events.amazonaws.com"
        }
        Action   = "sns:Publish"
        Resource = aws_sns_topic.security_alerts.arn
      }
    ]
  })
}
```

### Flow:

```
GuardDuty Finding (severity >= 7)
    ↓
EventBridge Rule (filter by severity)
    ↓
SNS Topic "security-alerts"
    ↓
Email → security-team@company.com
Lambda → Slack #security-incidents
```

## 5. Xử lý GuardDuty Findings

### Quy trình response:

```
1. DETECT — GuardDuty phát hiện finding
2. TRIAGE — Xác định severity + impact
3. INVESTIGATE — Xem chi tiết:
   - Affected resource (EC2 ID, IAM user...)
   - Source (IP, country, user agent)
   - Timeline (khi nào bắt đầu)
4. CONTAIN — Ngăn chặn:
   - Isolate EC2 (security group deny all)
   - Disable IAM user/access key
   - Block IP (NACL/WAF)
5. REMEDIATE — Khắc phục:
   - Patch vulnerability
   - Rotate credentials
   - Update security rules
6. REVIEW — Xem lại:
   - Root cause analysis
   - Update detection rules
   - Improve prevention
```

### Archive vs Suppress:

| Action | Mô tả | Khi nào |
|---|---|---|
| **Archive** | Đánh dấu đã xử lý | Đã investigate xong |
| **Suppress** | Tự động archive findings matching filter | Known false positive |

```
  ✅ Anonymous access to K8s API
  ✅ Exec into pod (kubectl exec)

  ✅ Exposed K8s dashboard
  ✅ Tor exit node accessing cluster
```
